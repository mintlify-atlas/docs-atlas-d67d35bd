---
title: "Text to Image"
description: "Generate images from a text prompt using POST /sdapi/v1/txt2img."
---

Send a POST request to `/sdapi/v1/txt2img` to generate one or more images from a text prompt. The server processes the request synchronously and returns base64-encoded images in the response body.

<Note>
  The API blocks until generation is complete. For long-running jobs, poll `GET /sdapi/v1/progress` in a separate thread to monitor status.
</Note>

## Request body

### Prompt

<ParamField body="prompt" type="string" default='""'>
  The positive prompt describing what you want to generate. Supports attention syntax (`(word:1.3)`) and textual inversion embedding tokens.
</ParamField>

<ParamField body="negative_prompt" type="string" default='""'>
  Text describing what to exclude from the generated image.
</ParamField>

<ParamField body="styles" type="array of strings" default="[]">
  List of style names to apply. Styles are defined in the WebUI and append to the prompt and negative prompt.
</ParamField>

### Seed

<ParamField body="seed" type="integer" default="-1">
  Seed for the random number generator. Set to `-1` to use a random seed. Use a fixed seed to reproduce results.
</ParamField>

<ParamField body="subseed" type="integer" default="-1">
  Seed for variation. Set to `-1` to use a random subseed.
</ParamField>

<ParamField body="subseed_strength" type="float" default="0">
  Influence of the subseed on the result. `0` means the subseed has no effect; `1` means the subseed fully overrides the main seed.
</ParamField>

<ParamField body="seed_resize_from_h" type="integer" default="-1">
  When set alongside `seed_resize_from_w`, resizes the seed noise to fit the given source dimensions before generation.
</ParamField>

<ParamField body="seed_resize_from_w" type="integer" default="-1">
  Width component for seed resize. See `seed_resize_from_h`.
</ParamField>

### Sampler and scheduler

<ParamField body="sampler_name" type="string" default="null">
  Name of the sampler to use (e.g., `"Euler a"`, `"DPM++ 2M"`, `"DDIM"`). Takes precedence over the deprecated `sampler_index`. Use `GET /sdapi/v1/samplers` to list available samplers.
</ParamField>

<ParamField body="scheduler" type="string" default="null">
  Name of the noise schedule to use (e.g., `"Automatic"`, `"Karras"`, `"Exponential"`). Use `GET /sdapi/v1/schedulers` to list available schedulers.
</ParamField>

### Generation parameters

<ParamField body="batch_size" type="integer" default="1">
  Number of images to generate in a single forward pass. All images in the batch share the same seed (offset by position unless `subseed_strength` is used).
</ParamField>

<ParamField body="n_iter" type="integer" default="1">
  Number of sequential batches to run. Total images generated = `batch_size × n_iter`.
</ParamField>

<ParamField body="steps" type="integer" default="50">
  Number of diffusion sampling steps. More steps generally improves quality but increases generation time.
</ParamField>

<ParamField body="cfg_scale" type="float" default="7.0">
  Classifier-free guidance scale. Higher values make the output follow the prompt more strictly. Typical range: `3`–`20`.
</ParamField>

<ParamField body="width" type="integer" default="512">
  Output image width in pixels. Must be a multiple of 8.
</ParamField>

<ParamField body="height" type="integer" default="512">
  Output image height in pixels. Must be a multiple of 8.
</ParamField>

<ParamField body="restore_faces" type="boolean" default="null">
  Apply face restoration after generation. Requires a face restorer (GFPGAN or CodeFormer) to be installed. Inherits from the options value when `null`.
</ParamField>

<ParamField body="tiling" type="boolean" default="null">
  Generate a seamlessly tiling image. Useful for textures and patterns.
</ParamField>

<ParamField body="eta" type="float" default="null">
  Eta parameter for some samplers (e.g., DDIM). Controls the stochasticity of the sampling process.
</ParamField>

### Sampler fine-tuning

<ParamField body="s_churn" type="float" default="null">
  Churn parameter for stochastic samplers. Inherits from options when `null`.
</ParamField>

<ParamField body="s_tmin" type="float" default="null">
  Minimum sigma at which to apply churn. Inherits from options when `null`.
</ParamField>

<ParamField body="s_tmax" type="float" default="null">
  Maximum sigma. Inherits from options when `null`.
</ParamField>

<ParamField body="s_noise" type="float" default="null">
  Noise multiplier for stochastic samplers. Inherits from options when `null`.
</ParamField>

<ParamField body="s_min_uncond" type="float" default="null">
  Minimum unconditional guidance sigma (used by some advanced sampling methods).
</ParamField>

### Highres fix

Enable the highres fix to generate a low-resolution base image and then upscale it in a second pass. This produces sharper high-resolution outputs than generating directly at large sizes.

<ParamField body="enable_hr" type="boolean" default="false">
  Enable the highres fix. When `true`, generation runs in two passes: a low-res first pass followed by an upscaled second pass.
</ParamField>

<ParamField body="denoising_strength" type="float" default="0.75">
  How much to change the image during the highres fix second pass. `0` preserves the original; `1` completely redraws it. Only used when `enable_hr` is `true`.
</ParamField>

<ParamField body="hr_scale" type="float" default="2.0">
  Upscale factor for the highres fix. The final image dimensions are `width × hr_scale` by `height × hr_scale`. Ignored when `hr_resize_x` or `hr_resize_y` are set.
</ParamField>

<ParamField body="hr_upscaler" type="string" default="null">
  Name of the upscaler to use in the highres fix pass (e.g., `"Latent"`, `"ESRGAN_4x"`, `"R-ESRGAN 4x+"`). Use `GET /sdapi/v1/upscalers` to list available upscalers.
</ParamField>

<ParamField body="hr_second_pass_steps" type="integer" default="0">
  Number of sampling steps for the highres fix second pass. When `0`, uses the same value as `steps`.
</ParamField>

<ParamField body="hr_resize_x" type="integer" default="0">
  Target width for the highres fix. When both `hr_resize_x` and `hr_resize_y` are `0`, `hr_scale` determines the output size.
</ParamField>

<ParamField body="hr_resize_y" type="integer" default="0">
  Target height for the highres fix. When only one of `hr_resize_x`/`hr_resize_y` is set, the other dimension is scaled to preserve aspect ratio.
</ParamField>

<ParamField body="hr_checkpoint_name" type="string" default="null">
  Checkpoint to use for the highres fix second pass. Defaults to the same checkpoint as the first pass when `null` or set to `"Use same checkpoint"`.
</ParamField>

<ParamField body="hr_sampler_name" type="string" default="null">
  Sampler to use for the highres fix second pass. Defaults to the main sampler when `null`.
</ParamField>

<ParamField body="hr_scheduler" type="string" default="null">
  Noise scheduler to use for the highres fix second pass.
</ParamField>

<ParamField body="hr_prompt" type="string" default='""'>
  Prompt to use for the highres fix second pass. When empty, the main `prompt` is used.
</ParamField>

<ParamField body="hr_negative_prompt" type="string" default='""'>
  Negative prompt for the highres fix second pass. When empty, the main `negative_prompt` is used.
</ParamField>

### Override settings

<ParamField body="override_settings" type="object" default="{}">
  A key-value map of option names to override for this request only. For example, `{"sd_model_checkpoint": "v1-5-pruned.safetensors"}` temporarily switches the model. Keys correspond to the fields returned by `GET /sdapi/v1/options`.
</ParamField>

<ParamField body="override_settings_restore_afterwards" type="boolean" default="true">
  When `true`, any values changed by `override_settings` are restored to their previous values after the request completes.
</ParamField>

### Refiner (SDXL)

<ParamField body="refiner_checkpoint" type="string" default="null">
  Checkpoint name for the SDXL refiner. Leave `null` to skip the refiner pass.
</ParamField>

<ParamField body="refiner_switch_at" type="float" default="null">
  Step fraction at which to switch from the base model to the refiner. Value between `0` and `1` (e.g., `0.8` switches at 80% of steps).
</ParamField>

### Scripts

<ParamField body="script_name" type="string" default="null">
  Name of a selectable script to run (e.g., `"X/Y/Z plot"`, `"Prompt matrix"`). Use `GET /sdapi/v1/scripts` to list available scripts.
</ParamField>

<ParamField body="script_args" type="array" default="[]">
  Arguments to pass to the selectable script specified in `script_name`. The order and types of arguments depend on the specific script.
</ParamField>

<ParamField body="alwayson_scripts" type="object" default="{}">
  Arguments for always-on scripts, keyed by script name. Each value is an object with an `args` array.

  ```json
  {
    "ControlNet": {
      "args": [{ "enabled": true, "model": "control_v11p_sd15_canny" }]
    }
  }
  ```
</ParamField>

### Output control

<ParamField body="send_images" type="boolean" default="true">
  When `true`, generated images are included in the response as base64-encoded strings. Set to `false` to skip image data in the response (useful when only saving to disk).
</ParamField>

<ParamField body="save_images" type="boolean" default="false">
  When `true`, generated images are saved to the configured output directory on the server.
</ParamField>

<ParamField body="force_task_id" type="string" default="null">
  Assign a specific task ID to this generation job. When `null`, the server generates a unique ID automatically.
</ParamField>

<ParamField body="infotext" type="string" default="null">
  A generation infotext string (as produced in the `info` response field). When provided, any field present in the infotext that is not already set in the request is populated from it. Useful for reproducing a generation from its metadata.
</ParamField>

---

## Response

<ResponseField name="images" type="array of strings">
  Base64-encoded images. The number of images equals `batch_size × n_iter`. When `send_images` is `false`, this array is empty.

  <Note>
    The encoding format (PNG, JPEG, or WebP) is controlled by the `samples_format` option in the server's settings.
  </Note>
</ResponseField>

<ResponseField name="parameters" type="object">
  A copy of the full request parameters as received by the server. Useful for confirming effective values including defaults.
</ResponseField>

<ResponseField name="info" type="string">
  A JSON string containing generation metadata: seeds used, model name and hash, sampler, steps, prompt, and other parameters. Parse this string to extract per-image seeds.

  <Expandable title="Example info fields">
    ```json
    {
      "prompt": "a cat",
      "all_prompts": ["a cat"],
      "negative_prompt": "",
      "seed": 42,
      "all_seeds": [42],
      "subseed": 3456789012,
      "all_subseeds": [3456789012],
      "subseed_strength": 0,
      "width": 512,
      "height": 512,
      "sampler_name": "Euler a",
      "cfg_scale": 7.0,
      "steps": 20,
      "batch_size": 1,
      "n_iter": 1,
      "sd_model_name": "v1-5-pruned-emaonly",
      "sd_model_hash": "cc6cb27103",
      "infotexts": ["a cat\nSteps: 20, Sampler: Euler a, CFG scale: 7, Seed: 42, Size: 512x512, Model: v1-5-pruned-emaonly"]
    }
    ```
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash curl
curl -X POST http://127.0.0.1:7860/sdapi/v1/txt2img \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a photorealistic cat sitting on a windowsill, golden hour lighting",
    "negative_prompt": "blurry, low quality, ugly",
    "steps": 25,
    "cfg_scale": 7,
    "width": 512,
    "height": 512,
    "sampler_name": "DPM++ 2M",
    "scheduler": "Karras",
    "seed": -1
  }'
```

```python python
import requests
import base64
from pathlib import Path

response = requests.post(
    "http://127.0.0.1:7860/sdapi/v1/txt2img",
    json={
        "prompt": "a photorealistic cat sitting on a windowsill, golden hour lighting",
        "negative_prompt": "blurry, low quality, ugly",
        "steps": 25,
        "cfg_scale": 7,
        "width": 512,
        "height": 512,
        "sampler_name": "DPM++ 2M",
        "scheduler": "Karras",
        "seed": -1,
    },
)
response.raise_for_status()
data = response.json()

# Save the first generated image
image_bytes = base64.b64decode(data["images"][0])
Path("output.png").write_bytes(image_bytes)

# Extract the seed used (useful for reproducibility)
import json
info = json.loads(data["info"])
print("Seed used:", info["seed"])
```

</CodeGroup>

### Highres fix example

<CodeGroup>

```bash curl
curl -X POST http://127.0.0.1:7860/sdapi/v1/txt2img \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a sprawling fantasy city, intricate architecture, highly detailed",
    "negative_prompt": "blurry, watermark",
    "steps": 25,
    "cfg_scale": 7,
    "width": 512,
    "height": 512,
    "enable_hr": true,
    "hr_scale": 2.0,
    "hr_upscaler": "R-ESRGAN 4x+",
    "hr_second_pass_steps": 15,
    "denoising_strength": 0.5
  }'
```

```python python
import requests
import base64
from pathlib import Path

response = requests.post(
    "http://127.0.0.1:7860/sdapi/v1/txt2img",
    json={
        "prompt": "a sprawling fantasy city, intricate architecture, highly detailed",
        "negative_prompt": "blurry, watermark",
        "steps": 25,
        "cfg_scale": 7,
        "width": 512,
        "height": 512,
        "enable_hr": True,
        "hr_scale": 2.0,
        "hr_upscaler": "R-ESRGAN 4x+",
        "hr_second_pass_steps": 15,
        "denoising_strength": 0.5,
    },
)
response.raise_for_status()
image_bytes = base64.b64decode(response.json()["images"][0])
# Result is a 1024×1024 image
Path("output_hires.png").write_bytes(image_bytes)
```

</CodeGroup>

### Temporary model switch example

Use `override_settings` to generate with a different model without changing the global setting:

<CodeGroup>

```bash curl
curl -X POST http://127.0.0.1:7860/sdapi/v1/txt2img \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "anime girl, vibrant colors",
    "steps": 20,
    "override_settings": {
      "sd_model_checkpoint": "anything-v5.safetensors"
    },
    "override_settings_restore_afterwards": true
  }'
```

```python python
import requests

response = requests.post(
    "http://127.0.0.1:7860/sdapi/v1/txt2img",
    json={
        "prompt": "anime girl, vibrant colors",
        "steps": 20,
        "override_settings": {
            "sd_model_checkpoint": "anything-v5.safetensors",
        },
        "override_settings_restore_afterwards": True,
    },
)
response.raise_for_status()
```

</CodeGroup>
