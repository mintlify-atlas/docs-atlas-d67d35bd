## Starting the server

<Tabs>
  <Tab title="Linux / macOS">
    Open a terminal in the `stable-diffusion-webui` directory and run:

    ```bash
    bash webui.sh
    ```

    On first launch, the script creates a Python virtual environment, installs PyTorch and all dependencies, and clones the required model repositories. This can take several minutes depending on your connection speed. Subsequent launches skip the installation steps and start much faster.
  </Tab>

  <Tab title="Windows">
    Open the `stable-diffusion-webui` folder in Windows Explorer and double-click **`webui-user.bat`**. Run it as a normal user — do not use "Run as administrator".

    On first launch, the script installs all dependencies automatically. A console window shows the progress.
  </Tab>
</Tabs>

When the server is ready, you will see output similar to:

```
Running on local URL:  http://127.0.0.1:7860
```

## Generating your first image

<Steps>
  <Step title="Open the web UI">
    Navigate to [http://127.0.0.1:7860](http://127.0.0.1:7860) in your browser. The **txt2img** tab opens by default.

    <Note>
      If the UI does not load, check the console window for errors. The server may still be initializing — wait for the `Running on local URL` line to appear before opening the browser.
    </Note>
  </Step>

  <Step title="Load a checkpoint model">
    A checkpoint model (`.safetensors` or `.ckpt` file) is required to generate images.

    **If no model is present**, the web UI automatically downloads Stable Diffusion 1.5 on first launch unless you pass `--no-download-sd-model`. You can also place a model file manually:

    1. Download a model from [Hugging Face](https://huggingface.co/models?pipeline_tag=text-to-image) or [Civitai](https://civitai.com).
    2. Place the file in the `models/Stable-diffusion/` directory inside your web UI folder.
    3. Click the **refresh** icon next to the checkpoint dropdown in the top-left of the UI.
    4. Select your model from the dropdown.

    <Tip>
      `.safetensors` format is preferred over `.ckpt` because it cannot execute arbitrary code when loaded.
    </Tip>
  </Step>

  <Step title="Write a prompt">
    In the **Prompt** field (the large text box at the top), describe what you want to generate. The model pays attention to emphasis using parentheses:

    - `a cat in a garden` — plain prompt
    - `a cat in a ((lush garden))` — extra attention on "lush garden"
    - `a cat in a (garden:1.4)` — explicit attention weight

    In the **Negative prompt** field below, list things you do not want to appear in the image. Common entries:

    ```
    blurry, low quality, extra limbs, deformed, watermark
    ```
  </Step>

  <Step title="Set generation parameters">
    The default values produce good results for most prompts, but you can adjust:

    | Parameter | Default | Description |
    |-----------|---------|-------------|
    | **Sampling method** | Euler a | The algorithm used to denoise the image. DPM++ 2M Karras and Euler a are popular starting points. |
    | **Sampling steps** | 20 | More steps generally produce higher quality but take longer. 20–30 is a good range. |
    | **Width / Height** | 512 × 512 | Output image dimensions. Keep dimensions as multiples of 8. Larger sizes require more VRAM. |
    | **CFG Scale** | 7 | How strictly the model follows the prompt. Higher values follow the prompt more closely but can produce artifacts. |
    | **Seed** | -1 | A value of `-1` picks a random seed each time. Set a specific number to reproduce the same image. |

    <Note>
      If you have less than 8 GB of VRAM, add `--medvram` to `COMMANDLINE_ARGS` in `webui-user.bat` or `webui-user.sh` before launching. For 4 GB or less, use `--lowvram`.
    </Note>
  </Step>

  <Step title="Generate">
    Click the **Generate** button (or press **Ctrl+Enter**). A progress bar appears below the button while the image is being created.

    The generated image appears on the right side of the screen. Below the image, the web UI displays the exact parameters used — including the seed — so you can reproduce or iterate on the result.
  </Step>

  <Step title="Save and review your output">
    Generated images are automatically saved to `outputs/txt2img-images/` inside the web UI directory, organized by date.

    The **parameters block** shown below the image in the UI includes:

    - The full prompt and negative prompt
    - Model name and hash
    - Sampling method and step count
    - CFG Scale, seed, and image dimensions

    You can drag any saved PNG back into the **PNG Info** tab to restore all its generation parameters into the UI.
  </Step>
</Steps>

## Useful launch options

You can pass command-line arguments by setting `COMMANDLINE_ARGS` in `webui-user.bat` (Windows) or `webui-user.sh` (Linux/macOS).

```bash title="webui-user.sh"
export COMMANDLINE_ARGS="--medvram --xformers"
```

```bat title="webui-user.bat"
set COMMANDLINE_ARGS=--medvram --xformers
```

| Argument | Effect |
|----------|--------|
| `--medvram` | Reduces VRAM usage at a small performance cost. Use with 4–8 GB GPUs. |
| `--lowvram` | Further reduces VRAM usage at a larger performance cost. Use with GPUs under 4 GB. |
| `--xformers` | Enables xFormers memory-efficient attention. Increases generation speed on supported NVIDIA GPUs. |
| `--listen` | Binds the server to `0.0.0.0` so other devices on your network can access the UI. |
| `--port 7860` | Sets the port (default is 7860). |
| `--autolaunch` | Opens the UI in your default browser automatically when the server starts. |
| `--api` | Enables the REST API alongside the web UI. |
| `--no-half` | Disables float16 precision. Use if you see black or corrupted images. |
| `--skip-torch-cuda-test` | Skips the GPU availability check at startup. |

<Warning>
  Using `--listen` exposes the UI to your local network. Do not use it on untrusted networks without also setting `--gradio-auth username:password`.
</Warning>
